Set up a Storybook

A running Storybook gives your agent live context about your components and design tokens — so the UI it builds matches your system.

connect a storybook with these options
point at a Storybook that's already running.
Enter the URL of a Storybook you already have running, local or remote. Cathode connects to it directly so your agent can reference your live design system.
run the Storybook already in your project.
Cathode starts your project's Storybook server and wires it up automatically. Your agent then has live access to every component and token in your design system.
build it from a Figma design.
Paste a Figma file URL and your agent recreates its components and stories through the Figma MCP. It reproduces the layout, colors, type, and assets as faithfully as it can.
scaffold one for your project's components.
Your agent installs Storybook for your project's framework and writes stories for your existing components. Leave it on auto-detect, or pick a framework to guide it.

Describe the app you want and your agent scaffolds it from scratch. It starts the dev server and opens the result here in the Browser.

Paste a Figma file or frame link and your agent rebuilds it as a running web app. It reads the frames over the Figma MCP and recreates them with your design system.

Paste a Git URL and your agent clones the repo, installs its dependencies, and starts the dev server. When it's live, the app opens in the Browser.
